CHAPTER 4: Checking File Status

The git status command shows which changes have been staged, which haven’t, and which files aren't being tracked by Git.

Remember, Git operates only on tracked files. To track a file, run:

git add <filename>

For a concise view, try:

git status -s

Status Codes:

  • ?? → Untracked file
  • A → File added to staging
  • M → Modified file
  • D → Deleted file
  • R → Renamed file
  • U → Unmerged conflict
  • MM → Staged + modified again
